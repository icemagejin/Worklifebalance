# 实施方案1：Agent 执行 VL 检查

## 当前状态

✅ 脚本已支持命令行参数
✅ 可以动态调整参数重新生成
⚠️ VL 检查功能暂不可用

## 问题分析

我（Aesthetics Agent）目前**没有直接的视觉理解能力**：
- 无法用 VL 模型分析本地图片
- 无法检测文字重叠、布局合理性等视觉问题

## 可用的替代方案

### 方案A：基于 PIL 的基本检查（当前可用）
```python
# 检查深色像素比例（文字是否太少）
dark_pixels = sum(1 for pixel in pixels if sum(pixel[:3]) < 600)
dark_ratio = dark_pixels / len(pixels)

if dark_ratio < 0.05:
    print("警告：文字区域太少")
```

**优点**：立即可用
**缺点**：无法检测视觉质量问题（重叠、遮挡、布局）

### 方案B：通过飞书群发送图片获取反馈（需用户配合）
1. 我生成海报 → 发送到美学群
2. 用户用 VL 工具检查 → 反馈问题
3. 我根据反馈调整参数 → 重新生成

**优点**：可以获得真实的视觉反馈
**缺点**：需要用户参与，不能完全自动化

### 方案C：等待 VL 能力接入（长期方案）
- 配置 VL_API_KEY（豆包 Vision Pro）
- 通过 API 调用 VL 模型
- 完全自动化检查

**优点**：真正的视觉理解
**缺点**：需要 API 配置

## 建议实施路径

### 短期（现在）
1. 使用 PIL 基本检查
2. 通过飞书群发送，用户手动验证
3. 根据用户反馈调整

### 中期（用户配合）
1. 用户提供 VL API Key
2. 配置到环境变量
3. 脚本自主调用 VL 检查

### 长期（平台升级）
- 等待平台提供 VL 能力
- 或者配置豆包 Vision Pro API
